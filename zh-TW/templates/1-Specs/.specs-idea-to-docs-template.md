# 規劃文件通用模板

> 此模板提供各類規劃文件的基本結構，AI 可根據實際需求增加欄位
> ⚠️ **所有規劃文件中的任務完成標記必須使用 `[✓]`，嚴禁使用 `[x]`**

---

## ⚠️ 資料夾結構規範

**正確的資料夾結構：**

```
[project]/1-Specs/[YYYY-MM-DD]-[plan-xxxxx]-[feature-name]/
  ├── [plan-xxxxx]-IDEA_DESCRIPTION.md (必須)
  ├── [plan-xxxxx]-CLEANUP_AND_INTEGRATION.md (必須)
  └── [plan-xxxxx]-[其他補充文件].md (至少一個)
```

**實際範例：**

```
your-project/1-Specs/2025-12-01-[plan-Ax4m2]-feature-name/
  ├── [plan-Ax4m2]-IDEA_DESCRIPTION.md
  ├── [plan-Ax4m2]-CLEANUP_AND_INTEGRATION.md
  ├── [plan-Ax4m2]-API_DESIGN.md
  └── [plan-Ax4m2]-THIRD_PARTY_INTEGRATION.md
```

**❌ 常見錯誤：**

- 建立名為 `plan/` 的資料夾（錯誤！應該是在現有的 `1-Specs/` 內建立子資料夾）
- 建立 `[project]/plan/[date]-[plan]-[feature]/`（錯誤！應該是 `1-Specs/` 而非 `plan/`）
- 忘記加上日期前綴
- 在錯誤的位置建立資料夾

**✅ 正確步驟：**

1. 確認專案已有 `1-Specs/` 資料夾
2. 在 `1-Specs/` 資料夾內建立 `[YYYY-MM-DD]-[plan-xxxxx]-[feature-name]/` 子資料夾
3. 在該子資料夾內建立所有規劃文件

### 🔀 並行性分析（建議加入）

**非強制，但建議在補充文件中加入並行性考量：**

```markdown
## 🔀 獨立性分析

### 可並行開發的部分

- **項目 A**：獨立功能，無依賴
- **項目 B**：獨立功能，無依賴
- **項目 C**：依賴 A 的介面定義，介面定義後可並行

### 高阻塞性項目（建議優先處理）

- **核心項目 X**：其他項目都依賴此項目，需最優先完成
- **共用部分 Y**：多處使用，建議優先建立

### 建議執行順序

1. 優先處理：[高阻塞性項目]
2. 並行處理：[獨立項目列表]
3. 後續處理：[依賴項目]
```

## 📝 使用指南

### 給 AI 的指引

1. **使用基本結構**
   - 每個文件都應包含基本欄位（文件類型、日期、概述）
   - 保持文件的一致性

2. **根據需求擴充**
   - AI 可以根據實際需求增加章節
   - 不要受限於模板，靈活調整

3. **優先考量並行性**
   - 建議在文件中加入「獨立性分析」章節
   - 幫助後續任務拆解 AI 規劃並行任務

4. **規劃任務數量限制**
   - 意識到此 Spec 最終會被拆分為 1-8 個任務。
   - **上限：** 至多不超過 8 項任務。
   - **原則：** 如果不是絕對複雜的任務，應盡量保持簡單的任務數量。

### 模板使用原則

- ✅ 使用基本結構確保一致性
- ✅ 根據需求自由擴充欄位
- ✅ 加入並行性考量（建議）
- ✅ 保持文件清晰易讀
- ❌ 不要死板遵循模板
- ❌ 不要過度複雜化
