# Phase-Priority 匹配規則 - 快速參考卡

**用途：** AI 和使用者的快速參考指南  
**完整指引：** 請參考 [PHASE_PRIORITY_GUIDELINES.md](./PHASE_PRIORITY_GUIDELINES.md)

---

## 🎯 核心原則（必須記住！）

```
Phase 編號 ≈ 執行順序 ≈ 優先級降低
```

### Phase 與 Priority 對應表

| Phase        | 適合優先級                      | 任務特性             |
| ------------ | ------------------------------- | -------------------- |
| **Phase 1**  | 🔴 緊急 + 🟠 核心高優先級       | 基礎建設、核心功能   |
| **Phase 2**  | 🟠 中高優先級 + 🟡 核心中優先級 | 服務層、業務邏輯     |
| **Phase 3**  | 🟡 中優先級 + 🟢 低優先級       | UI 層、使用者介面    |
| **Phase 4+** | 🟢 低優先級 + 💡 建議執行       | 整合測試、優化、文件 |

---

## ⚠️ 特殊情況

**如果任務因依賴必須在最後階段，但優先級為緊急：**

✅ **必須做：**

1. 在 TASK_OVERVIEW 表格中標註「⚠️ 最終階段關鍵任務」
2. 在任務描述中說明原因
3. 確保所有前置任務優先級足夠高

❌ **禁止做：**

- 不標註就直接將緊急任務放在 Phase 4
- 前置任務優先級過低

---

## ✅ 快速驗證（完成任務分解後）

- [ ] Phase 1 包含所有緊急任務？（特殊情況除外）
- [ ] Phase 4 主要是低優先級任務？（特殊情況除外）
- [ ] 特殊情況已明確標註？
- [ ] Phase 之間優先級逐步降低？

---

## 📖 選擇任務時的優先順序

**從 `2-Plans/` 選擇任務時：**

1. **優先級優先：** 🔴 緊急 > 🟠 高 > 🟡 中 > 🟢 低 > 💡 建議
2. **同優先級內：** Phase 1 > Phase 2 > Phase 3 > Phase 4+
3. **依賴關係：** 確保前置任務已完成

**範例：**

- ✅ 優先：Phase 1 的緊急任務
- ✅ 次之：Phase 1 的高優先級任務
- ✅ 再次：Phase 2 的緊急任務（如果有特殊情況）
- ✅ 再次：Phase 2 的高優先級任務

---

## 🚫 常見錯誤

❌ **錯誤 1：** Phase 4 出現大量緊急任務  
✅ **正確：** 緊急任務應在 Phase 1-2，除非有不可避免的依賴

❌ **錯誤 2：** Phase 1 都是低優先級任務  
✅ **正確：** Phase 1 應該是緊急或高優先級任務

❌ **錯誤 3：** UI 任務在 Phase 4  
✅ **正確：** UI 任務通常在 Phase 3
