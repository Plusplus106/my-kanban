# 階段 5: 專案歸檔 (7-Done → 8-Archived)

> **📌 重要提醒:**
>
> **開始使用本提示詞前,請務必先閱讀 [KANBAN_INSTRUCTION.md](../../KANBAN_INSTRUCTION.md)**
>
> 本階段必須遵循的核心原則:
>
> - 📋 檔案移動規則
> - 📋 停下動作等待確認原則

---

## 🎯 目標

當一個專案的所有實作計畫都已完成時，將整個專案的規格文件和已完成計畫一起歸檔到 `[專案名稱]/8-Archived/[YYYY-MM-DD]-[spec-xxxxx]-[feature-name]/`，建立專案摘要，並清理原始文件資料夾。

---

## 📖 第一步：閱讀規則與確認歸檔條件

### 1. 閱讀規則文件

- `templates/7-Done/DONE_RULES.md`
- `templates/8-Archived/ARCHIVED_RULES.md`

### 2. 掃描所有專案規格文件

讀取 `[專案名稱]/1-Specs/` 中的所有文件，列出可能需要歸檔的專案。

### 3. 向用戶確認要歸檔的專案

---

## ✅ 第二步：檢查專案完成狀態

### 1. 讀取規格文件並提取計畫列表

確認 `1-Specs` 中的需求是否都已轉化為計畫。

### 2. 檢查 7-Done/ 中的已完成計畫

確認所有相關計畫都已完成。

### 3. 檢查其他資料夾中的未完成計畫

必須確認 `2-Plans` ~ `6-On-hold` 中沒有此專案的計畫。

### 4. 執行歸檔條件檢查

- [ ] 所有計畫都已完成
- [ ] 所有測試都已通過
- [ ] 專案功能已驗證
- [ ] 每份 done 計畫都有 Testing 品質閘門證據（測試程式碼、測試指令、all green 輸出）

---

## 📁 第三步：建立歸檔資料夾結構

### 1. 建立歸檔資料夾

```
[專案名稱]/8-Archived/YYYY-MM-DD-[spec-xxxxx]-[feature-name]/
├── specs/            # 規格文件
└── 7-Done-plans/     # 已完成計畫
```

### 2. 複製規格文件

將 `1-Specs` 中的文件複製到 `specs/`。

### 3. 移動已完成計畫

將 `7-Done` 中的計畫移動到 `7-Done-plans/`。

---

## 📊 第四步：建立專案歸檔摘要

建立 `YYYY-MM-DD-專案名稱-summary.md` 文件。

### 摘要內容

- 專案名稱、期間、歸檔時間
- 專案描述與成果
- 統計數據 (Feature/Bug/Test 數量)
- 技術棧
- 學習經驗

---

## 🧹 第五步：清理所有中間階段資料夾（強制）

### 1. 清理所有中間階段的殘留

歸檔完成後，必須確認以下所有中間階段中**已無此專案的任何殘留文件或空資料夾**：

- `[專案名稱]/1-Specs/[專案資料夾名稱]/` → 刪除
- `[專案名稱]/2-Plans/[對應資料夾]/` → 刪除
- `[專案名稱]/3-Progressing/[對應資料夾]/` → 刪除
- `[專案名稱]/4-Testing/[對應資料夾]/` → 刪除
- `[專案名稱]/5-Re-testing/[對應資料夾]/` → 刪除（如有）
- `[專案名稱]/6-On-hold/[對應資料夾]/` → 刪除（如有）
- `[專案名稱]/7-Done/[對應資料夾]/` → 刪除

### 2. 清空 4-Testing/temp/ 測試資料

清空測試資料，但保留資料夾結構。

### 3. 驗證清理完成

使用以下指令確認無殘留：

```bash
find [專案名稱]/{1-Specs,2-Plans,3-Progressing,4-Testing,5-Re-testing,6-On-hold,7-Done}/ -name "*[spec-id]*" -o -name "*[plan-id]*"
```

若有殘留，立即清除。歸檔完成後，該批次的文件應**僅存在於 `8-Archived/`**。

---

## 📝 第六步：提供歸檔完成報告

提供詳細的歸檔報告，確認歸檔成功。

⏸️ 停下動作，等待進一步指示。
