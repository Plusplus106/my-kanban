# 階段 2: 開始實作計畫 (2-Plans → 3-Progressing)

> **📌 重要提醒:**
>
> **開始使用本提示詞前,請務必先閱讀 [KANBAN_INSTRUCTION.md](../../KANBAN_INSTRUCTION.md)**
>
> 本階段必須遵循的核心原則:
>
> - 📋 程式碼註解必須使用英文
> - 📋 檔案移動規則
> - 📋 停下動作等待確認原則
> - 📋 暫存檔案管理 (使用 temp/ 資料夾)
> - ⭐ **Lint 檢查必須通過**

---

## 🎯 目標

從 `[專案名稱]/2-Plans/` 挑選一個計畫，移動到 `[專案名稱]/3-Progressing/`，並開始撰寫程式碼。

**注意：本階段專注於程式碼開發，不包含測試撰寫（測試在下一階段處理）。**

---

## 📖 第一步：閱讀規則與挑選計畫

### 1. 閱讀規則文件

在開始前，請先閱讀：

- `templates/3-Progressing/PROGRESSING_RULES.md` - 了解開發流程和規範
- `templates/2-Plans/PLANS_RULES.md` - 了解計畫卡片結構

### 2. 掃描可執行的計畫

請列出 `[專案名稱]/2-Plans/` 中所有符合以下條件的計畫：

- **無依賴關係**，或
- **依賴已滿足**（被依賴的計畫已在 `[專案名稱]/7-Done/` 中）

**檢查「是否可並行執行」欄位:**

- 查看哪些計畫標記為「是」
- 這些計畫可以同時由多個 AI 工具並行處理
- 讀取完 Plan 文件並準備開始實作時，優先檢查是否可以同步開始實作
- 若任務彼此可並行且無依賴衝突，開啟同步實作模式
- 若無法同步開始實作，必須依前後依賴關係逐步執行任務內容

### 3. 計畫優先級排序

按以下順序排列可執行計畫：

1. `...-1-urgent-*` - 緊急計畫
2. `...-2-high-*` - 高優先級計畫
3. `...-3-mid-*` - 中優先級計畫
4. `...-4-low-*` - 低優先級計畫

**⭐ 重要提醒：優先級評估原則**

- 優先級是基於**阻塞性和重要性**
- 會阻塞其他計畫的基礎設施計畫應該是高優先級

### 4. 向用戶確認

**列出建議要處理的計畫（前 3-5 個），並等待用戶確認。**

---

## 🔄 第二步：移動並更新計畫卡片（⚠️ 僅處理文件，不開始編寫程式碼）

**⭐ 重要：本步驟只負責計畫文件的移動和更新，不要開始變更任何程式碼！**

確認計畫後：

### 1. 移動檔案

將計畫檔案從 `2-Plans/` **移動（mv）**到 `3-Progressing/`

**🧹 來源清理（強制）：**

- 移動後，確認 `2-Plans/` 中該批次文件已不存在
- 若來源資料夾已為空，刪除空資料夾

### 2. 更新計畫卡片

**必須修改以下欄位：**

```markdown
**狀態:** 處理中 (Progressing)

**開始處理時間:** [填入實際當前時間]

**負責的 AI 工具 1:** [AI 名稱]

**開發進度:**

- [ ] 計畫文件更新完成
- [ ] 程式碼框架建立
- [ ] 核心邏輯實現
- [ ] 錯誤處理
- [ ] Lint 檢查通過
- [ ] 完成開發

**相關程式碼檔案:**

- (開發過程中陸續新增)
```

### 3. 生成 Commit 訊息

為這次計畫文件的變更生成一個英文的 commit 訊息。

### 4. ⏸️ **停下動作，等待用戶確認**

**❌ 不要繼續進行程式碼開發，必須等待用戶回覆「繼續」！**

---

## 💻 第三步：開始開發（⚠️ 需要用戶確認後才能執行）

**⭐ 重要：只有在用戶確認「繼續」後，才能開始本步驟！**

### 開發要求

#### ✅ 程式碼品質要求

1. **清晰易讀**
2. **註解規範** (必須使用英文)
3. **錯誤處理**
4. **Lint 檢查** (必須通過)

#### 📝 開發過程中持續更新

1. **更新「相關程式碼檔案」欄位**
2. **更新「開發進度」checklist**

### ⚠️ 開發過程中的注意事項

- **不要撰寫測試程式碼**：此階段只負責功能開發
- **遇到問題時**：先詢問用戶或參考文件
- **暫存檔案**：放到 `4-Testing/temp/`

---

## ✅ 第四步：開發完成檢查

### 1. Lint 檢查（⭐ 必須執行）

所有程式碼在完成後必須通過 Lint 檢查。

### 2. 自我檢查清單

- [ ] Lint 檢查已通過
- [ ] 所有功能都已實現
- [ ] 程式碼註解完整（使用英文）
- [ ] 「相關程式碼檔案」欄位已更新

### 3. 更新計畫卡片

- 完成所有開發進度項目
- 填寫 AI 工具結束時間

### 4. 🚪 Stage Exit Checkpoint（強制）

移動前必須確認：

- 所有「開發進度」checkbox 已勾選為 `[✓]`（包含「Lint 檢查通過」、「完成開發」）
- 已補齊「本次實作結果」描述
- 若有 PLAN_OVERVIEW，已更新統計與狀態

### 5. 移動計畫

將計畫從 `3-Progressing/` 移動到 `4-Testing/`
**注意：不要移到 `7-Done/`！**
**注意：移入 `4-Testing/` 後，不可只停在文件移動；下一步必須完成測試程式碼與 all green 驗證。**

### 6. 🧹 來源清理（強制）

移動到 `4-Testing/` 後：

- 確認 `3-Progressing/` 中該批次文件已不存在
- 若來源資料夾已為空，刪除空資料夾

---

## 📊 第五步：提供開發摘要

完成後，提供詳細的開發摘要，包含實作內容、使用技術、建立的檔案等。

⏸️ 停下動作，等待確認後再處理下一個計畫或進入測試階段。
